Project Overview

This project implements a semantic search and visualization system for Plasma Physics papers from ArXiv. Unlike traditional keyword-based search, this system uses dense vector embeddings from Sentence-BERT to find conceptually similar papers based on meaning. The system processes paper abstracts, generates 384-dimensional embeddings, and enables fast similarity search using FAISS indexing. Interactive visualizations created with UMAP reveal the structure of the research landscape, showing how different plasma physics topics cluster together in the embedding space.

Key Challenges

  • Processing LaTeX-heavy academic text while preserving semantic meaning
  • Scaling embedding generation and search to 10,000+ papers efficiently
  • Creating meaningful 2D projections that reveal research cluster structure

Technical Implementation

The pipeline consists of four main stages: data collection from the ArXiv API, text preprocessing to handle LaTeX and special characters, embedding generation using the all-MiniLM-L6-v2 model, and index building with FAISS for sub-millisecond similarity search. The UMAP algorithm projects the high-dimensional embedding space into 2D for visualization while preserving both local and global structure.

Model Architecture

The embedding model (all-MiniLM-L6-v2) is a distilled version of BERT optimized for semantic similarity tasks. It processes the concatenated title and abstract of each paper, producing a normalized 384-dimensional vector. These vectors are indexed using FAISS's IndexFlatIP for exact inner-product (cosine) similarity search, enabling query times under 1ms even for 10,000+ documents.

C++ Reference Implementation

As an educational exercise, I also implemented a pure C++17 version of the sentence embedder from scratch, without any external dependencies beyond the standard library. This demonstrates how transformer-based neural networks work at the lowest level—implementing everything from WordPiece tokenization to multi-head self-attention.

Implementation Highlights

  • Zero dependencies — Pure C++17 with only the standard library (no BLAS, no SIMD)
  • Complete transformer stack — Tensor operations, layer normalization, GELU activation, multi-head attention
  • WordPiece tokenizer — Full implementation matching the HuggingFace tokenizer behavior
  • Python bindings — ctypes-based interface for integration with the main Python pipeline
  • CLI tool — Standalone command-line interface for embedding text

Results and Impact

The system successfully indexes over 10,000 Plasma Physics papers with embedding generation at 87 papers/second on CPU. Search queries return results in under 1ms with meaningful semantic similarity—papers about similar topics consistently appear together in search results, even when they share no common keywords.

The UMAP visualizations reveal clear clustering of research topics: papers on tokamak physics, laser-plasma interactions, and space plasma phenomena form distinct regions in the 2D projection. The average category overlap between query papers and their nearest neighbors is 58.9%, indicating strong semantic coherence.

The interactive HTML visualization enables researchers to explore the paper landscape, zoom into clusters of interest, and discover related work they might have missed through traditional keyword search.

Technical Overview

Data Pipeline

Fetches paper metadata from ArXiv's OAI-PMH API with rate limiting and caching. Text preprocessing removes LaTeX formatting while preserving semantic content.

Key Components:

  • ArXiv API client with exponential backoff
  • LaTeX-aware text preprocessing
  • Parquet storage for efficient I/O

Search System

FAISS index enables exact nearest-neighbor search in sub-millisecond time. Normalized embeddings allow cosine similarity via inner product.

Key Features:

  • Sentence-BERT embeddings (384-dim)
  • FAISS IndexFlatIP for exact search
  • CLI tool for interactive exploration
